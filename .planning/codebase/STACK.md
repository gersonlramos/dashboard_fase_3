# Technology Stack

**Analysis Date:** 2026-03-25

## Languages

**Primary:**
- Python 3.11 — all application code (dashboard, data extraction scripts, automation)

## Frameworks & Libraries

**Web UI:**
- `streamlit` 1.55.0 — interactive web dashboard served on port 8501
  - Entry point: `app/dashboard/dashboard.py`
  - Launch command: `streamlit run dashboard.py --server.enableCORS false --server.enableXsrfProtection false`

**Data Visualization:**
- `plotly` 5.24.1 — interactive charts via `plotly.express` and `plotly.graph_objects`

**Data Processing:**
- `pandas` 2.2.3 — CSV loading, DataFrame manipulation, date arithmetic
- `numpy` 2.2.1 — numerical operations (sigmoid/learning curve calculations in `app/dashboard/dashboard.py`)

**HTTP / Networking:**
- `requests` 2.32.3 — all HTTP calls to Jira REST API (HTTPBasicAuth)
- `urllib3` 2.2.3 — SSL warning suppression for corporate network (`verify=False` pattern across all scripts)

**Configuration:**
- `python-dotenv` 1.0.0 — loads `EMAIL` and `API_TOKEN` from `.env` file in local dev

## Build Tools

- None — no build step; scripts run directly with the Python interpreter
- No webpack, no frontend bundler — Streamlit handles all UI serving

## Package Management

- **pip** — dependencies declared in `requirements.txt` at project root
- Lockfile: absent (only pinned versions in `requirements.txt`, no `pip.lock`)
- Dev container installs via `pip3 install --user -r requirements.txt`

## Runtime Environment

- **Python** 3.11 (pinned in `.devcontainer/devcontainer.json` and `.github/workflows/atualizar_dados.yml`)
- **Container image:** `mcr.microsoft.com/devcontainers/python:1-3.11-bookworm` (Debian Bookworm)
- **Streamlit server:** port 8501, CORS and XSRF protection disabled for Codespaces preview
- No virtual environment tooling (venv/Poetry/Pipenv) beyond plain pip
- No linting or formatting config detected (no `.flake8`, `.pylintrc`, `pyproject.toml`, `black.toml`)

## Data Storage

- No database — all persistent data stored as CSV files under `app/dados/`
- CSV files are committed to the repository and updated automatically by CI
- Key data files:
  - `app/dados/FASE_3.csv` — main subtask summary (generated by `app/scripts/script_atualizacao.py`)
  - `app/dados/processos_seguintes.csv` — epics list
  - `app/dados/datas_esperadas_por_lake.csv` — planned dates per data lake
  - `app/dados/pendencias_BF3E4-293.csv` — pending items
  - `app/dados/historico_BF3E4-293.csv` — status history
  - `app/dados/historico/historico_completo-*.csv` — per-epic history (9 files, one per business domain)

## Key Configuration Files

- `requirements.txt` — pinned Python dependencies
- `app/dashboard/.streamlit/config.toml` — Streamlit theme (dark base, brand colors `#1f77b4`, `#0d1b2a`)
- `.devcontainer/devcontainer.json` — dev container setup for GitHub Codespaces / VS Code
- `.github/workflows/atualizar_dados.yml` — GitHub Actions CI schedule for automated Jira data refresh

## Platform Requirements

**Development:**
- Python 3.11
- pip with `requirements.txt`
- Optional: GitHub Codespaces (pre-configured via `.devcontainer/devcontainer.json`)

**Production / Deployment:**
- GitHub Codespaces (primary deployment target as configured in devcontainer)
- Port 8501 exposed and auto-forwarded
- Automated data refresh runs on GitHub Actions (Ubuntu Latest runner, scheduled Mon–Fri at 4 daily intervals)
- No Dockerfile or docker-compose for standalone containerized deployment
