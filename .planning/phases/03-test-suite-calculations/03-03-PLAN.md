Create `tests/test_smoke.py` using Streamlit's AppTest API (available since Streamlit 1.28, present in 1.55.0):

```python
import os
import pytest
import streamlit as st
from streamlit.testing.v1 import AppTest

DASHBOARD_PATH = os.path.join(
    os.path.dirname(__file__), '..', 'app', 'dashboard', 'dashboard.py'
)

@pytest.fixture(autouse=True)
def clear_cache():
    """Clear st.cache_data before and after each test to avoid contamination."""
    st.cache_data.clear()
    yield
    st.cache_data.clear()


def test_dashboard_loads_without_exception():
    """Smoke test: the full dashboard runs without raising any exception."""
    at = AppTest.from_file(DASHBOARD_PATH, default_timeout=60)
    at.run()
    assert not at.exception, f"Dashboard raised an exception: {at.exception}"
```

Key notes:

- `AppTest.from_file` runs the script in a sandboxed Streamlit context — `st.set_page_config()` executes normally without crashing
- `at.exception` is `None` when the app runs cleanly; otherwise it holds the exception
- `default_timeout=60` gives enough time for all CSV reads on first run
- The `clear_cache` fixture is `autouse=True` so it runs for every test in the file
- The real CSV files (FASE_3.csv, etc.) are present in the project — no mocking needed for this smoke test
